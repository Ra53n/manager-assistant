# Manager assistant

Простое нативное Mac-приложение (SwiftUI): чат с нейросетью **DeepSeek**.
Окно с историей сообщений, поле ввода и отправка по Enter / кнопке.

## 1. Задать API-ключ

⚠️ Ключ **не хранится в коде** и не попадает в git. Приложение читает его во время
выполнения. Положи свой ключ в локальный файл:

```bash
mkdir -p ~/.config/manager-assistant
echo "sk-..." > ~/.config/manager-assistant/deepseek.key   # ключ с https://platform.deepseek.com
```

Альтернатива — переменная окружения (приоритетнее файла):

```bash
export DEEPSEEK_API_KEY="sk-..."
```

Ключ создаётся в личном кабинете DeepSeek (раздел **API Keys**), баланс нужно пополнить.
Пока ключ не задан — приложение запустится, но при отправке покажет понятную подсказку.

## 2. Запуск во время разработки

Из папки проекта:

```bash
swift run
```

Откроется окно чата. Полный Xcode не обязателен — достаточно Command Line Tools.
Если стоит Xcode, можно открыть `Package.swift` в Xcode и нажать **Run** (⌘R).

## 3. Сборка готового приложения (.app)

```bash
bash run.sh
open ManagerAssistant.app
```

Скрипт соберёт release-бинарь и упакует его в `ManagerAssistant.app` — обычное Mac-приложение.

## Параметры генерации (на каждый чат)

Кнопка **⚙︎ (slider.horizontal.3)** в правом верхнем углу чата открывает настройки.
Параметры хранятся **отдельно для каждого чата**:

| Параметр | Диапазон | Что делает |
|----------|----------|------------|
| `temperature` | 0…2 | Креативность/случайность ответа |
| `top_p` | 0…1 | Nucleus sampling |
| `max_tokens` | 256…8192 | Максимальная длина ответа |
| `stop` | до 16 строк | Стоп-последовательности (через запятую) |
| Формат ответа | Текст / JSON | DeepSeek `response_format` (`text` / `json_object`) |
| Роль ассистента | текст | Системная роль/задача (системный промпт) |
| Условия завершения | список | Что ассистент должен собрать перед завершением |
| Действие при завершении | текст | Что сделать, когда все условия собраны |

DeepSeek **не поддерживает** `top_k`, `frequency_penalty`, `presence_penalty` — API их
игнорирует, поэтому в настройках их нет. У reasoning-моделей (`deepseek-reasoner`)
`temperature` и `top_p` также не действуют.

### Сценарные условия завершения

Роль + условия + действие собираются в системный промпт ([`PromptBuilder`](Sources/ManagerAssistant/Models.swift)).
Модель задаёт по одному уточняющему вопросу, пока не соберёт все условия, затем выполняет
действие. Пример (приём заказа в Макдональдсе):

- **Роль:** «Ты принимаешь заказ в Макдональдсе»
- **Условия:** «Какой напиток», «Нужна ли картошка», «Хочет ли пожертвовать детям»
- **Действие:** «сформируй и выпиши итоговый заказ»
- **Формат ответа:** JSON — итог придёт структурированным объектом

При формате **JSON** все ответы модели возвращаются как JSON-объект (включая промежуточные),
поэтому для диалога с естественными вопросами выбирай **Текст**, а JSON — когда нужен
машинно-читаемый результат.

## Структура

| Файл | Назначение |
|------|------------|
| `Config.swift` | Чтение API-ключа (env/файл), endpoint, модель |
| `Models.swift` | Чат, сообщение, настройки генерации + DTO запроса/ответа |
| `DeepSeekClient.swift` | HTTP-запрос к DeepSeek (async/await) |
| `ChatViewModel.swift` | Список чатов, выбор, отправка, настройки |
| `ContentView.swift` | UI: сайдбар чатов, сообщения, лист параметров |
| `App.swift` | Точка входа + иконка в Dock |

## Дальнейшие шаги (вне первого этапа)

Стриминг ответов, хранение ключа в Keychain, сохранение истории между запусками,
выбор модели в UI.
