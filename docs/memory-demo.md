# Проверка модели памяти — сценарий видео + бенчмарк-промпты

Файл для демонстрации, что ТЗ выполнено: явная модель памяти из 3 слоёв,
раздельное хранение, явный выбор «что и куда», проверка попадания данных и
влияния на ответы.

## 0. Соответствие ТЗ → реализация

| Требование ТЗ | Где в приложении |
|---|---|
| Краткосрочная (текущий диалог) | `Chat.memory` → внутри **chats.json** |
| Рабочая (данные задачи) | `Project.entries` (секции) → **projects.json** |
| Долговременная (профиль, решения, знания) | глобальные `MemoryItem` → **memory.json** |
| Разные типы хранятся ОТДЕЛЬНО | **три разных файла** (см. выше) |
| Явно выбираю, что и куда | кнопки на сообщении «В память» (выбор уровня) и «В проект» (секция); панели «мозг» и «папка» |
| Какие данные попадают в каждый слой | инспекция файлов (ниже) + панели |
| Как влияет на ответы | сравнение вкл/выкл инъекции (ниже) |

Папка данных:
`~/Library/Application Support/ManagerAssistant/`

## Инспекция слоёв (показывать на экране во время видео)

```bash
cd ~/Library/Application\ Support/ManagerAssistant

# ДОЛГОВРЕМЕННАЯ (профиль/знания) — глобально
python3 -m json.tool memory.json

# РАБОЧАЯ (проекты и их секции)
python3 -c "import json;[print('•',p['title'],'→ секции:',[e['title'] for e in p['entries']]) for p in json.load(open('projects.json'))]"

# КРАТКОСРОЧНАЯ (заметки внутри конкретных чатов)
python3 -c "
import json
for c in json.load(open('chats.json')):
    if c.get('memory'): print('•',c['title'][:30],'→',[m['text'] for m in c['memory']])
"
```

Совет для записи: держи терминал рядом и **повторяй команду после каждого
сохранения** — видно, как запись появляется именно в своём файле.

## Сценарий видео (≈4–6 минут)

Персона для связности: «Костя — Android-разработчик, проект ЭХО (macOS-приложение
для транскрибации встреч), любит краткие ответы по пунктам».

### Часть 1. Краткосрочная память (текущий диалог)

1. Вкладка **Чаты** → новый чат. Промпт:
   > В этом диалоге фокус только на экране импорта аудио. Ничего другого пока не трогаем.
2. Наведи на своё сообщение → **«В память»** → Уровень: **Краткосрочная**, Тип:
   Заметка → Сохранить. *(показать выбор уровня — это «явно выбираю, что и куда»)*
3. Проверка хранения: кнопка **«мозг»** → секция «Краткосрочная»; в терминале —
   запись появилась в **chats.json** (поле `memory` этого чата).
4. Влияние на ответ (тот же чат):
   > Напомни, на чём мы сегодня сфокусированы?
   → ассистент отвечает «экран импорта аудио».
5. Доказательство изоляции: новый чат → тот же вопрос → **не знает** (краткосрочная
   память не утекает в другие диалоги).

### Часть 2. Рабочая память (проект, cowork)

1. Вкладка **Проекты** → **+** → Название: `ЭХО — транскрибатор встреч`,
   Инструкции: `macOS-приложение для транскрибации аудио встреч. Стек: Swift, Core Data, движок Whisper. Отвечай развёрнуто.`
2. В созданном диалоге проекта — промпт:
   > Спроектируй модель данных (сущности, поля, связи) для хранения встреч и их транскриптов с таймкодами.
   → получаем подробный ответ. Т.к. у проектных диалогов **автосекции включены**,
   полный ответ сам уходит секцией в проект.
3. Проверка: кнопка **«папка»** (или ползунки у проекта) → «Секции проекта» —
   появилась секция целиком; в терминале — **projects.json** вырос.
4. Влияние / общая память проекта (cowork): в проекте **«Новый диалог»** → промпт:
   > Какие сущности мы определили в модели данных и почему?
   → новый диалог **знает** (память проекта общая для всех его диалогов).
5. Контраст: вкладка **Чаты**, обычный чат → тот же вопрос → **не знает** (он вне
   проекта).
6. Финал части: в панели проекта — **«Собрать»** → ассистент сшивает все полные
   секции в единый документ.

### Часть 3. Долговременная память (профиль, решения, знания)

1. В любом чате — промпт:
   > Запомни про меня: я Android-разработчик, всегда отвечай мне кратко, по пунктам, на русском.
2. Наведи на сообщение → **«В память»** → Уровень: **Долговременная**, Тип:
   Предпочтение → Сохранить.
3. Проверка: кнопка «мозг» → «Долговременная»; терминал — **memory.json**.
4. Влияние (персонализация во ВСЁМ приложении): **новый чат** (хоть в другом
   проекте) → промпт:
   > Объясни, что такое dependency injection.
   → ответ **краткий, по пунктам, на русском** (профиль подмешался).
5. Контраст (как память влияет на ответ): в этом чате ⚙︎ → секция «Память» →
   выключи **«Долговременная память»** → тот же вопрос → ответ становится более
   общим/длинным. Включи обратно — снова персонализированный.

### Часть 4. Раздельное хранение + явный выбор (сводно, 20 сек)

- Терминал: три команды из раздела «Инспекция слоёв» — видно три **разных файла**,
  в каждом своё. Это «разные типы хранятся отдельно».
- Открой лист «В память» — пикер **Уровень** (Долговременная / Краткосрочная) и
  «В проект» — это «вы явно выбирали, что и куда».

### Часть 5 (опционально). Автоматический режим

- ⚙︎ → «Память» → включи **«Ассистент памяти»** + **«Автозапись»**.
- Промпт: `Кстати, я работаю в команде Avito Android и предпочитаю Kotlin.`
- После ответа модель **сама** извлечёт профиль в долговременную (без кнопок) —
  показать новую запись в memory.json. (Без «Автозаписи» — она кладёт это в
  подсказки на подтверждение: счётчик на «мозге».)

## Бенчмарк-промпты (копировать)

Короткий набор для быстрой проверки каждого слоя:

```
[Краткосрочная]
SET:  В этом диалоге фокус только на экране импорта аудио.
CHECK: Напомни, на чём мы сегодня сфокусированы?

[Рабочая / проект]
SET:  Спроектируй модель данных для хранения встреч и транскриптов с таймкодами.
CHECK (новый диалог в проекте): Какие сущности мы определили в модели данных?
CHECK (обычный чат вне проекта): тот же вопрос → должен НЕ знать.

[Долговременная]
SET:  Запомни: я Android-разработчик, отвечай кратко по пунктам, на русском.
CHECK (новый чат): Объясни, что такое dependency injection.
       → форма ответа должна следовать профилю (кратко, пунктами, на русском).
TOGGLE-OFF (⚙︎ → Память → Долговременная = выкл): тот же вопрос → ответ обобщённее.
```

## Что говорить про «как это влияет на ответы»

Память подставляется в **системный промпт** каждого запроса (stateless API не
хранит контекст сам). Поэтому:
- долговременная меняет ответы во ВСЕХ чатах (персонализация);
- рабочая — во всех диалогах своего проекта (общий контекст задачи);
- краткосрочная — только в своём диалоге.
Выключение тумблера инъекции в настройках убирает слой из промпта — ответ заметно
меняется, что и доказывает влияние слоя.
